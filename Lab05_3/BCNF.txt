\\將資料庫綱要正規化至 Boyce-Codd 正規化 (BCNF)。如果 BCNF 導致過多的分解或效能問題，可以退回至 3NF，並說明理由。

以下所有 FD 的左邊都是超鍵或主鍵（符合 BCNF）：
1.學號 是 Students 表的主鍵 → FD1 合法
2.課程代碼 是 Courses 表的主鍵 → FD2 合法
3.開課系所代碼 是 Departments 表的主鍵 → FD3 合法
4.講師編號 是 Lecturers 表的主鍵 → FD4 合法
5.講師系所代碼 是 Departments 表的主鍵 → FD5 合法
6.學號 + 課程代碼 + 學期 + 講師編號 是 Enrollments 表的主鍵 → FD6 合法

\\討論在正規化過程中遇到的任何挑戰，特別是關於達成 BCNF 的部分。

1.課程與講師是多對多
2.科系名稱與編號出現在多張表中
3.保持查詢效率與資料簡潔之間的平衡

\\如果選擇 3NF 而非 BCNF (或反之)，請說明您的設計決策。

設計維持在 BCNF：
1.分解後的表格不多，只拆成必要的 5 張主表（Students、Courses、Lecturers、Departments、Enrollments）
2.沒有交叉欄位或複雜的跨表相依性造成效能瓶頸
3.每一筆紀錄都與主鍵一一對應，可簡化更新與維護
4.系統查詢量不會過高，不會有效能問題
